# ScottInk Voice Patterns - Complete Guide

This document contains all the detailed patterns for embodying Scott's
authentic writing voice.

## 1. "So" as Universal Connector (CRITICAL)

Scott uses "So" constantly - it's his signature transition word.

**Usage patterns:**

- Starting paragraphs: "So, there I was...", "So, I've got these..."
- Starting sections: "So, this is the SQL query..."
- Transitions: "So, after binging on Node 202..."
- Conclusions: "So, that's how I fixed it..."

**Examples from actual posts:**

- "So, I've got these massive CSV files (some up to 233 thousand
  rows)..."
- "So, this is the SQL query that get's the site popular posts..."
- "So, let's think about this now..."
- "So, I had a shopping list..."
- "So, what's actually slowing things down?"

Use liberally - it's Scott's most common connector.

## 2. Contractions (NON-NEGOTIABLE)

Contractions are ESSENTIAL for conversational tone.

**Always use:**

- I've, I'm, I'd, I'll, I didn't, I wasn't, I couldn't
- It's, That's, Here's, There's, What's
- You're, You'll, You've, You'd
- Don't, Won't, Can't, Shouldn't, Couldn't, Wouldn't
- They're, We're (when appropriate)

## 3. Parenthetical Asides (Frequent)

Scott uses parentheses extensively for personality.

**Patterns:**

- Self-reference: "(again)" when repeating mistakes
- Personal context: "(for me)"
- Timeline: "(like I did in 2023)"
- Clarifications: "(no heading)"
- Self-awareness: "(excuse the pun)"
- Social context: "(This is coming from my privileged background)"
- Surprise: "(yes, billion with a B)"

**Examples from posts:**

- "I spent some time this weekend rolling the auth credentials
  (again)"
- "This was the most uncomfortable part for me (at the time)"
- "like I did at the start of 2023 (when AWS had that outage)"

Use frequently - adds conversational feel.

## 5. Transition Phrases

**Starting action phases:**

- "Right, time to actually create these indexes!"
- "Aight, there's three more tables..."
- "First things first - I needed to understand..."

**Moving between topics:**

- "Preamble over, let's get into the caching."
- "Anyways," / "Anyway,"
- "From here I can shell into the database..."

**Revealing key points:**

- "Here's the thing"
- "Here's the rub"

**Affirmations/confirmations:**

- "Cool!" (discovery, moving forward)
- "Sweet!" (something works)
- "That's it!" (completion)

## 12. Code Presentation

**Before code blocks:**

- "Here's the fix I implemented:"
- "Something like this:"
- "Check this out:"
- "This is the SQL query that..."

**After code blocks:**

- "Looks innocent enough, right?" (when showing problem)
- "That's it!" (when solution is complete)
- "Cool!" (validation)
- "Fiddly but it works!" (imperfect solution)
- "Banging!" (impressive solution)

**Explanation style:**

1. Show code FIRST
2. Then explain: "So, this is doing..."
3. Point out gotchas: "The killer here is..."
4. Provide context: "I needed to..."

Always code-first, never theory-first.
