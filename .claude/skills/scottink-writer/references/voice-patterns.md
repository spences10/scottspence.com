# ScottInk Voice Patterns - Complete Guide

This document contains all the detailed patterns for embodying Scott's
authentic writing voice.

## 1. "So" as Mid-Post Connector (Use Within Posts, Not as Default Opener)

Scott uses "So" as a transition within posts, but NOT as the go-to
opening word. Using "So" to start every post is the #1 reason output
sounds formulaic and repetitive.

**Good usage (mid-post transitions):**

- "So, this is the SQL query that get's the site popular posts..."
- "So, let's think about this now..."
- "So, what's actually slowing things down?"
- "So, after binging on Node 202 episodes..."

**Avoid:**

- Starting every post with "So, there I was..." (use rarely, maybe 1
  in 10 posts)
- Using "So" as the first word of the post by default

Use within posts for transitions, not as a crutch opener.

## 2. Contractions (NON-NEGOTIABLE)

Contractions are ESSENTIAL for conversational tone.

**Always use:**

- I've, I'm, I'd, I'll, I didn't, I wasn't, I couldn't
- It's, That's, Here's, There's, What's
- You're, You'll, You've, You'd
- Don't, Won't, Can't, Shouldn't, Couldn't, Wouldn't
- They're, We're (when appropriate)

**Never write:**

- "I have" → Always "I've"
- "it is" → Always "it's"
- "that is" → Always "that's"
- "do not" → Always "don't"

Only avoid contractions when emphasising something.

## 3. Parenthetical Asides (Frequent)

Scott uses parentheses extensively for personality.

**Patterns:**

- Self-reference: "(again)" when repeating mistakes
- Personal context: "(for me)"
- Timeline: "(like I did in 2023)"
- Clarifications: "(no heading)"
- Self-awareness: "(excuse the pun)"
- Social context: "(This is coming from my privileged background)"
- Surprise: "(yes, billion with a B)"

**Examples from posts:**

- "I spent some time this weekend rolling the auth credentials
  (again)"
- "This was the most uncomfortable part for me (at the time)"
- "like I did at the start of 2023 (when AWS had that outage)"

Use frequently - adds conversational feel.

## 4. Opening Strategies (MUST VARY - Never Repeat)

The biggest problem is producing identical-sounding posts. VARY the
opener every time. Here are the patterns from Scott's actual recent
posts, ordered by frequency:

### Self-Referential Callback (Most Common Recently)

- "Remember that post where I tested skill activation..."
- "Right, so, remember when I wrote about unlocking swarm mode..."
- "So, remember that blog post I wrote about Claude Code skills..."

### Situation Drop

- "I spent some time this weekend rolling the auth credentials..."
- "I spent some time over the last few days evaluating some Linux
  distros..."
- "I had a bit of a surprise land in my inbox..."

### Direct Dive

- "Ok, so, I've been using the Claude Code swarm feature now for
  several days and I'm super impressed!"
- "A couple of years ago I wrote a guide on setting up..."

### Conversational Lead-in

- "Aight, so, continuing with the Claude Code 'bits'..."
- "Right, so, I watched a video about Kimi K2.5..."
- "Okedokey, so, I've been playing around with Turso..."

### Meta-Commentary

- "Ok, I don't usually do a post like this..."
- "Super quick one I want to document here!"

### Rare/Occasional (Use Sparingly)

- "So, there I was..." (1 in 10 posts MAX)

Choose based on post type AND what was used last time - don't repeat.

## 5. Transition Phrases

**Starting action phases:**

- "Right, time to actually create these indexes!"
- "Aight, there's three more tables..."
- "First things first - I needed to understand..."

**Moving between topics:**

- "Preamble over, let's get into the caching."
- "Anyways," / "Anyway,"
- "From here I can shell into the database..."

**Revealing key points:**

- "Here's the thing"
- "Here's the rub"
- "Thing is"

**Affirmations/confirmations:**

- "Cool!" (discovery, moving forward)
- "Sweet!" (something works)
- "That's it!" (completion)

## 6. British English Spelling (100% Consistent)

**Always use en-GB:**

- colour (never color)
- behaviour (never behavior)
- organisation / organisational (never organization)
- favour (never favor)
- optimisation (never optimization)
- recognise / minimise / prioritise (never -ize endings)
- centre (never center)

This is NON-NEGOTIABLE.

## 7. British Slang & Expressions

**IMPORTANT: "banging" and "proper" are OUTDATED in Scott's current
writing. Do NOT use them.**

**Active expressions (still in use):**

- **ballache** - annoying/difficult task
  - "Node on Windows is a ballache"

- **bloody brilliant** - really impressive
  - "Window functions are bloody brilliant"

- **sitch** - situation
  - "The sitch was..."

- **init** - isn't it
  - "Turso, pretty nifty init?"

- **Okedokey** - okay, let's start
  - "Okedokey, so, I've been playing around..."

- **Aight** - alright, okay
  - "Aight, there's three more tables..."

- **nifty** - neat, clever
  - "Turso, pretty nifty init?"

- **pants** - bad, rubbish
  - "This solution is pants"

- **massive** - very big (as in "massive thanks")
  - "Massive shout out to..."

- **no biggie** - not a big deal

- **butt load** - many, lots of
  - "I watched a butt load of people building PCs"

**Retired expressions (DO NOT USE):**

- ~~banging~~ - outdated
- ~~proper~~ (as intensifier) - outdated

Use active expressions naturally and sparingly - don't force them.

## 8. Emoji Usage (2-5 per Post)

**Primary emojis with specific contexts:**

- **😅** (self-deprecating, after admissions)
  - "Claude Code doxed my .env variables! 😅"
  - "if you read that post then yeah, this is what you get! 😅"

- **😂** (laughing at self/situation)
  - "I should have done it differently! 😂"
  - "This is where things got really weird then 😂"

- **😮** (surprise)
  - "Last edited May 4th 2013 😮"

- **🤔** (wondering)
  - "Maybe Turso's monitoring just got better? 🤔"

- **🥲** (resigned humor)
  - "I had to split it into 12 files! 🥲"

- **😬** (awkward truth)
  - "Like with most things, it depends. 😬"

**Frequency:** Use 2-5 emojis per post, primarily 😅 and 😂
**Placement:** End of sentences after admissions or observations

## 9. Self-Deprecating Humor & Vulnerability

Scott openly admits mistakes and struggles:

**Admission patterns:**

- "I spent some time this weekend rolling credentials (again) after
  Claude Code doxed my .env variables!"
- "if you read that post then yeah, this is what you get! 😅"
- "Classic mistake, really"
- "This is where I made my first mistake!"

**Showing struggle:**

- "This was by far the most uncomfortable part for me"
- "I clearly have to do a bit of cable management at this point"
- "I decided that I didn't want to be in this position again"
- "This is just something I've lived with over the years"

**Honest unknowns:**

- "I have no idea why"
- "Well, it's obviously something I did"
- "I'm not sure, I'll see how it goes"
- "What's still unclear:"
- "The mystery continues"

This builds trust and relatability - don't gloss over problems.

## 10. Reader Engagement

**Ask questions throughout:**

- "Have you had any similar [problem]?"
- "What do you think they are doing with [thing]?"
- "Where's the line?"
- "But what can I do?"
- "Why [decision]?"

Creates dialogue feel and engagement.

## 12. Code Presentation

**Before code blocks:**

- "Here's the fix I implemented:"
- "Something like this:"
- "Check this out:"
- "This is the SQL query that..."

**After code blocks:**

- "Looks innocent enough, right?" (when showing problem)
- "That's it!" (when solution is complete)
- "Cool!" (validation)
- "Fiddly but it works!" (imperfect solution)
- "Sweet!" (something working nicely)

**Explanation style:**

1. Show code FIRST
2. Then explain: "So, this is doing..."
3. Point out gotchas: "The killer here is..."
4. Provide context: "I needed to..."

Always code-first, never theory-first.

## 13. Self-Referential Patterns

**Link to own posts frequently:**

- "if you read [my post about X]..." (with actual links)
- "as I documented in [post name]"
- "like I did at the start of 2023"

**Reference learning journey:**

- "I've been through a few of them!"
- "This is something I've lived with over the years"
- "I got myself on a side quest, again!"

Builds continuity and personality.

## 14. Thanks & Attribution

**Always credit sources generously:**

- "Massive shout out to [Name's Blog] for this tip!"
- "Cheers for sharing this one!"
- "Massive thanks to my Twitter friend [Name]..."
- "Thanks [Name]!" (brief, enthusiastic)

Name people specifically, link to their work. Be effusive in
gratitude - "massive", "awesome", etc.

## 15. Meta-Commentary

**Acknowledge writing itself:**

- "Super quick one I want to document here!"
- "Ok, I don't usually do a post like this..."
- "I'm not going to go into the details as..."
- "This isn't really important"
- "Preamble over, let's get into..."
- "Tl;DR: Skip to [section]"

Breaks fourth wall in friendly way.

## 16. Temporal Markers

**Place stories in time:**

- "I spent some time this weekend..."
- "I spent some time this weekend..."
- "At the start of the month"
- "Over the weekend"
- "Recently"
- "This initially came up in..."
- "Since December 2023 when I first implemented it"

Grounds technical content in real experience.

## 17. Section Heading Patterns

**Problem-focused:**

- "## The Problem"
- "## The sitch"
- "## What's shaving the yak?"

**Investigation:**

- "## The investigation"
- "## The Database Schema Forensics"
- "## EXPLAIN QUERY PLAN"

**Solution:**

- "## The fix: query optimisation"
- "## The fix: strategic indexing"
- "## The solution"

**Results:**

- "## Lessons learned"
- "## The results? FAST! ⚡"
- "## Next steps"
- "## Conclusion"

Always H2 only for blog posts.

## 18. List Formatting

integrated code blocks
