# Newsletter Format Guide

## Title Format

**Pattern:** `[Playful Title] - [Month Year]`

**Examples:**

- "The one where I broke SQLite - October 2025"
- "Ship it and iterate - September 2025"
- "Learning in public - August 2025"

## Required Structure

### 1. Opening Prose (No Heading)

Brief summary of the month's theme or what's in the newsletter.

### 2. Highlights Section

2-4 major accomplishments or topics from the month.

### 3. Technical Deep Dives

Detailed discussion of technical learnings or implementations.

### 4. Open Source (If Applicable)

Updates on open source contributions or projects.

### 5. What's Coming

Preview of upcoming work or plans.

### 6. Closing Prose (No Heading)

Brief wrap-up or reflection.

## Common Issues to Flag

### Red Flags (Fix Required)

- Links in headings (causes nested `<a>` tags in HTML)
- Third-person voice
- American spellings
- Vague statements without specifics
- Missing sections from required structure

### Yellow Flags (Review)

- Very short sections (under 50 words)
- No code references in technical newsletter
- Missing repository links
- Generic opening that doesn't capture the month

## Reporting Format

**Overall Assessment:** [Ready/Needs Changes/Hold]

**Issues to Address:**

- Priority items listed

**Ready to Publish:** Yes/No with brief reasoning
